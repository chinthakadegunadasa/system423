# පරිච්ඡේදය 03: චිත්ත වග්ගය (The Mind)
### මධ්‍යම සැකසුම් ඒකකය, කර්නල් ආරක්ෂාව සහ පද්ධති අධිභාරය (Kernel, Security & Overflow Logic)

#### 3.1 සිත: පද්ධති මෙහෙයුම් පද්ධතිය (The System OS)
ධම්මපදයේ චිත්ත වග්ගය මගින් විග්‍රහ කරන්නේ සිතේ පවතින අතිශය චපල සහ වේගවත් ස්වභාවයයි. *"ඵන්දනං චපලං චිත්තං"* (සිත වෙවුලන සුළුය, චපලය). 

* **පද්ධති විද්‍යාව:** සිත යනු පද්ධතියේ **Execution Thread** එකයි. එය අතිශය ඉහළ වේගයකින් (High Frequency) ක්‍රියාත්මක වන අතර, නිසි ලෙස පාලනය (Control Logic) නොකළහොත් පද්ධතිය අස්ථාවර (Unstable) වේ.
* **දමනය කළ සිත:** ඉංජිනේරු විද්‍යාවේදී **Damping Control** මගින් පද්ධතියක අනවශ්‍ය වෙවුලීම (Oscillation) පාලනය කරන්නාක් මෙන්, සිත දමනය කිරීමෙන් පද්ධතියේ ප්‍රතිදානයන් (Outputs) ප්‍රශස්ත කළ හැකිය.

#### 3.2 සිතේ ගෘහ නිර්මාණ ශිල්පය (Architecture of the Mind)
1. **දුරංගමං (Remote Processing):** සිත ඉතා ඈතට ගමන් කරයි. මෙය **Cloud Computing** ක්‍රියාවලියකට සමානය. සිරුර එක් ස්ථානයක තිබියදී සිත වෙනත් දත්ත පරාසයන් (Data Domains) වෙත පිවිසෙයි.
2. **ඒකචරං (Single-Threaded Execution):** සිත වරකට ගන්නේ එක් අරමුණක් පමණි. මෙය **Single-threaded CPU** එකක ක්‍රියාවලියට සමානය. අපට "බහු-කාර්ය" (Multi-tasking) ලෙස පෙනෙන්නේ සිතේ අතිශය වේගවත් මාරුවීම (Context Switching) නිසාය.
3. **අසරීරං (Virtual Environment):** සිතට භෞතික ස්වරූපයක් නැත. එය පද්ධතියේ **Virtual Instance** එකක් ලෙස ක්‍රියා කරයි.

#### 3.3 Root Access: සිතේ පරමාධිපත්‍යය (System Sovereignty)
පද්ධති ඉංජිනේරු විද්‍යාවේදී **Root Access** යනු පද්ධතියක සම්පූර්ණ පාලනය (Total Control) ලබා ගැනීමයි. ධම්මපදයේ පෙන්වා දෙන්නේ සාමාන්‍ය මිනිසා තම සිතේ 'Root' පරිශීලකයා නොවන බවයි; ඒ වෙනුවට සිත පාලනය කරන්නේ බාහිරින් එන කෙලෙස් (External Scripts/Malware) විසිනි.

* **පරමාධිපත්‍යය (Sovereignty):** සිත දමනය කිරීම යනු පද්ධතියේ මූලික කර්නලය (Kernel) බාහිර ආක්‍රමණයන්ගෙන් ආරක්ෂා කර ගැනීමයි. 
* **Privilege Escalation:** භාවනාව සහ සිහිය පුහුණු කිරීම යනු පද්ධතියේ සාමාන්‍ය පරිශීලක මට්ටමක සිට Administrator/Root මට්ටම දක්වා තම පාලන බලතල (System Privileges) වැඩි කර ගැනීමයි.

#### 3.4 System Security: මාරයා සහ Firewall Protocol
*"කුම්භූපමං කායමිමං විදිත්වා - නගරූපමං චිත්තමිදං ඨපෙත්වා"*
මෙම ගාථාව සෘජුවම **Network Security** සහ **Endpoint Protection** සංකල්පවලට සමාන වේ.

* **නගරූපමං චිත්තං (Fortified City):** සිත බලකොටුවක් ලෙස සකස් කර ගැනීම යනු ශක්තිමත් **Firewall** එකක් නිර්මාණය කිරීමයි. පද්ධතියේ අනවශ්‍ය 'Ports' (ඉන්ද්‍රියන්) වසා දැමීම මෙහිදී සිදුවේ.
* **යෝධේථ මාරං පඤ්ඤායුධේන (The Weapon of Wisdom):** මාරයා හෙවත් පද්ධතියට රිංගන 'Malware' සමඟ සටන් කළ යුත්තේ **ප්‍රඥාව** නැමැති අවියෙනි. ප්‍රඥාව යනු පද්ධතියේ සිදුවන දත්ත විකෘති කිරීම් (Code Injections) හඳුනා ගැනීමට ඇති **Intrusion Detection System (IDS)** එකයි.

#### 3.5 Buffer Overflow: පද්ධතිමය අධිභාරය (System Overload)
සිතේ අධික ලෙස සිතුවිලි ජනනය වීම පද්ධතියක **Buffer Overflow** තත්ත්වයකට සමානය. 

* **The Crash:** සිතේ "Buffer" එක යනු තාවකාලිකව දත්ත රඳවා තබා ගන්නා මතක ධාරිතාවයි. අප්‍රමාණ වූ ආශාවන් සහ බිය නිසා සිතුවිලි සීමාව ඉක්මවා ජනනය වන විට, පද්ධතියේ **Stack Memory** එක පිරී ගොස් පද්ධතිය "Crash" වේ.
* **Context Switching Overhead:** සිත එකවර අරමුණු රාශියක දිවීමට උත්සාහ කිරීම නිසා ඇතිවන ප්‍රලම්භනය (Lag) පද්ධතියේ කාර්යක්ෂමතාව අඩු කරයි. 
* **Garbage Collection (GC):** භාවනාව යනු සිතේ පද්ධතිය තුළ ක්‍රියාත්මක වන **Automatic Garbage Collection** ක්‍රියාවලියයි. අනවශ්‍ය දත්ත (Akusala Data) පිරිසිදු කිරීම මගින් පද්ධතියට අවශ්‍ය "Free Space" එක නැවත ලබා ගත හැකිය.

#### 3.6 Security & Performance Matrix

| තාක්ෂණික පරාමිතිය | ධර්ම සන්දර්භය | පද්ධතිමය භූමිකාව |
| :--- | :--- | :--- |
| **Root Access** | චිත්තස්ස දමථෝ | පද්ධතියේ පූර්ණ පාලනය තමා සතු කර ගැනීම. |
| **Firewall** | නගරූපමං චිත්තං | බාහිර බලපෑම් (Malware) වැළැක්වීම. |
| **Buffer Management** | සමාධි / ඒකග්ගතා | දත්ත අධිභාරය (Overflow) පාලනය කිරීම. |
| **Garbage Collection** | විදර්ශනා / පාරිශුද්ධිය | මතකය පිරිසිදු කර පද්ධතිය ස්ථාවර කිරීම. |
| **IDS (Intrusion Detection)** | පඤ්ඤා (ප්‍රඥාව) | හානිකර Code Injections හඳුනා ගැනීම. |
